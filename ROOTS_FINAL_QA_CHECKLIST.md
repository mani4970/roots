# Roots Final QA Checklist

이 체크리스트는 Roots 웹앱/PWA를 배포하거나 큰 수정 후 빠르게 검수하기 위한 최종 확인표입니다.
코드 파일이 아니라 운영/테스트용 문서입니다.

## 사용 방법

- 큰 패치 후에는 P0와 P1을 먼저 확인합니다.
- 문구/i18n만 바꾼 경우에는 관련 화면과 Vercel build 성공 여부를 우선 확인합니다.
- PWA 관련 수정 후에는 앱을 완전히 종료했다가 다시 실행해서 확인합니다.
- Supabase/RLS/SQL 수정 후에는 실제 화면에서 데이터가 보이는지, 다른 사용자의 비공개 데이터가 보이지 않는지 확인합니다.

---

## P0 - 반드시 확인

### 1. 배포 / 빌드

- [ ] Vercel build가 성공한다.
- [ ] 배포된 URL이 정상 접속된다.
- [ ] 모바일 브라우저에서 첫 화면이 정상 표시된다.
- [ ] PWA로 실행했을 때 흰 화면이나 무한 로딩이 없다.

### 2. 로그인 / 회원가입

- [ ] 이메일 로그인 가능.
- [ ] Google 로그인 가능.
- [ ] 로그아웃 가능.
- [ ] 로그아웃 후 보호 페이지 접근 시 로그인 화면으로 이동한다.
- [ ] 회원가입 화면이 정상 표시된다.
- [ ] 언어 선택 후 로그인/회원가입 화면 문구가 해당 언어로 유지된다.

### 3. 홈 화면

- [ ] 홈 화면이 정상 표시된다.
- [ ] 오늘의 말씀 영역이 정상 표시된다.
- [ ] 체크인/QT/기도/결단 카드가 정상 표시된다.
- [ ] 완료된 루틴은 완료 상태로 표시된다.
- [ ] 미완료 루틴은 진행 가능한 상태로 표시된다.
- [ ] 전체 루틴 완료 시 celebration/confetti가 정상 표시된다.

### 4. TreeGrowth / 낮밤 이미지

현재 기준:

```ts
NIGHT_START_HOUR = 19
NIGHT_END_HOUR = 6
```

- [ ] 19:00~05:59 현지 기기 시간에는 dark tree 이미지가 표시된다.
- [ ] 06:00~18:59 현지 기기 시간에는 daytime tree 이미지가 표시된다.
- [ ] 해외/다른 시간대에서도 서버 시간이 아니라 기기 현지 시간 기준으로 표시된다.

### 5. QT 작성

- [ ] QT 작성 화면에 진입 가능.
- [ ] 말씀 선택 화면이 정상 표시된다.
- [ ] 성경 본문 불러오기가 정상 작동한다.
- [ ] 여러 본문 추가가 가능하다.
- [ ] 6단계 QT 작성이 가능하다.
- [ ] 자유형식 QT 작성이 가능하다.
- [ ] 주일예배 QT 작성이 가능하다.
- [ ] 결단 입력이 가능하다.
- [ ] 임시저장 버튼이 작동한다.
- [ ] 임시저장 toast/alert가 현재 언어로 표시된다.
- [ ] 임시저장 toast/alert가 불필요하게 줄바꿈되지 않는다.
- [ ] 임시저장 후 다시 이어쓰기 시 저장된 내용이 복원된다.
- [ ] 이어쓰기 시 말씀 선택 화면이 잠깐 깜빡이지 않는다.
- [ ] QT 완료 후 완료 화면으로 이동한다.
- [ ] QT 완료 confetti가 화면 아래쪽에서 터지는 스타일로 표시된다.

### 6. QT 기록

- [ ] QT 탭이 정상 표시된다.
- [ ] 연도별 아코디언이 표시된다.
- [ ] 현재 연도는 기본으로 열린다.
- [ ] 과거 연도는 기본으로 접힌다.
- [ ] 현재 월은 기본으로 열린다.
- [ ] 이전 월은 기본으로 접힌다.
- [ ] 화살표로 연도/월을 열고 닫을 수 있다.
- [ ] QT 기록 상세 화면으로 이동 가능.
- [ ] 전체 복사 가능.
- [ ] QT 나누기 모달이 정상 표시된다.
- [ ] 전체 커뮤니티/그룹 공유 상태가 정상 표시된다.

### 7. 기도

- [ ] 기도 페이지가 정상 표시된다.
- [ ] 기도 중/응답된 기도 탭 전환 가능.
- [ ] 새 기도 제목 작성 가능.
- [ ] 기존 기도 제목 수정 가능.
- [ ] 기도 완료 처리 가능.
- [ ] 기도 응답 처리 가능.
- [ ] 중보기도 요청 가능.
- [ ] 기도 완료 시 celebration/confetti가 정상 표시된다.

### 8. 커뮤니티

- [ ] 커뮤니티 페이지가 정상 표시된다.
- [ ] 기도/QT/그룹 탭 전환 가능.
- [ ] QT 나눔 목록이 표시된다.
- [ ] 중보기도 목록이 표시된다.
- [ ] 함께 기도 버튼이 작동한다.
- [ ] prayer_count가 비정상적으로 중복 증가하지 않는다.
- [ ] QT 반응 버튼이 작동한다.

### 9. 그룹

- [ ] 그룹 목록이 정상 표시된다.
- [ ] 공개 그룹이 정상 표시된다.
- [ ] 참여 중인 그룹이 정상 표시된다.
- [ ] 비공개 그룹은 권한 없는 사용자에게 노출되지 않는다.
- [ ] 그룹 생성 가능.
- [ ] 그룹 초대 링크 공유 가능.
- [ ] 그룹 초대 링크로 참여 가능.
- [ ] 그룹 즐겨찾기 별표 클릭 가능.
- [ ] 즐겨찾기한 그룹이 위로 올라온다.
- [ ] PWA를 완전히 종료 후 다시 실행해도 별표가 유지된다.
- [ ] 새 QT 나눔이 있는 그룹에 새 글 표시가 뜬다.
- [ ] 그룹 안에서 아직 안 읽은 QT에 안 읽음 표시가 뜬다.
- [ ] 그룹에 들어가면 새 글이 읽음 처리된다.
- [ ] 그룹 나가기 버튼이 보인다.
- [ ] 그룹 나가기 확인 모달이 Roots 스타일로 표시된다.
- [ ] 그룹 나가기 후 목록 상태가 정상 갱신된다.

### 10. 계정 삭제

- [ ] 프로필에서 계정 삭제 UI가 표시된다.
- [ ] 삭제 확인 흐름이 명확하다.
- [ ] 테스트 계정으로 삭제 시 앱 데이터가 삭제된다.
- [ ] 테스트 계정으로 삭제 시 Supabase Auth user가 삭제된다.
- [ ] 삭제 후 `/welcome`으로 이동한다.
- [ ] 삭제 후 기존 계정으로 다시 로그인되지 않는다.

## P1 - 강력 권장 확인

### 11. 언어 / i18n

확인 언어:

- [ ] 한국어
- [ ] Deutsch
- [ ] English
- [ ] Français

주요 화면:

- [ ] Welcome
- [ ] Login
- [ ] Signup
- [ ] Home
- [ ] QT
- [ ] QT Write
- [ ] QT Record
- [ ] Prayer
- [ ] Community
- [ ] Join
- [ ] Profile
- [ ] Onboarding
- [ ] Check-in Result

확인 항목:

- [ ] 문구가 한국어로 고정되어 있지 않다.
- [ ] 독일어/프랑스어 긴 문장이 버튼 밖으로 심하게 넘치지 않는다.
- [ ] toast/alert 문구가 현재 언어로 표시된다.
- [ ] 공유 메시지가 현재 언어로 표시된다.

### 12. PWA

- [ ] 홈 화면에 추가된 PWA가 정상 실행된다.
- [ ] 로그인 세션이 유지된다.
- [ ] 언어 설정이 유지된다.
- [ ] 테마 설정이 유지된다.
- [ ] 그룹 즐겨찾기가 유지된다.
- [ ] 앱 완전 종료 후 재실행해도 주요 상태가 유지된다.
- [ ] 아이콘이 정상 표시된다.
- [ ] PWA 설치 유도 팝업은 표시되지 않는다.

### 13. Supabase / RLS

- [ ] 주요 테이블 RLS가 enabled 상태다.
- [ ] private prayer는 다른 사용자에게 보이지 않는다.
- [ ] private group은 멤버가 아닌 사용자에게 보이지 않는다.
- [ ] group_members 정보가 관련 없는 사용자에게 넓게 노출되지 않는다.
- [ ] prayer_count는 클라이언트 직접 update가 아니라 RPC를 통해 처리된다.
- [ ] service role key는 Vercel에서 Sensitive로 설정되어 있다.
- [ ] service role key가 `NEXT_PUBLIC_` 이름으로 존재하지 않는다.

### 14. 환경변수

Vercel Production에 필요한 값:

- [ ] `NEXT_PUBLIC_SUPABASE_URL`
- [ ] `NEXT_PUBLIC_SUPABASE_ANON_KEY`
- [ ] `SUPABASE_SERVICE_ROLE_KEY`

삭제/불필요:

- [ ] `ANTHROPIC_API_KEY` 또는 `CLAUDE_API_KEY`가 더 이상 필요 없다면 제거되어 있다.
- [ ] `NEXT_PUBLIC_`로 시작하는 Claude/Anthropic key가 없다.

### 15. Git / 파일 정리

- [ ] `.gitignore`가 프로젝트 루트에 있다.
- [ ] `.env` / `.env.local`이 Git에 올라가지 않는다.
- [ ] `.next/`가 Git에 올라가지 않는다.
- [ ] `node_modules/`가 Git에 올라가지 않는다.
- [ ] `.DS_Store`가 Git에 올라가지 않는다.
- [ ] `public/confetti.json`은 더 이상 사용하지 않으므로 삭제 가능하다.

## P2 - 필요할 때 확인

### 16. Capacitor 전환 전 확인

- [ ] OAuth redirect URL을 web/native로 분리할 계획이 있다.
- [ ] Supabase redirect URL에 앱 scheme을 추가할 계획이 있다.
- [ ] localStorage 사용은 `lib/clientStorage.ts` 중심으로 정리되어 있다.
- [ ] Capacitor 전환 시 `@capacitor/preferences`로 바꾸기 쉬운 구조다.
- [ ] Service Worker를 native 앱에서 계속 쓸지 web-only로 제한할지 결정한다.
- [ ] 앱 아이콘/splash screen 준비.
- [ ] iOS/Android deep link 준비.

### 17. 나중으로 보류한 작업

- [ ] `profiles` 공개 조회 정리 / `public_profiles` view 작업은 보류.
- [ ] profiles 테이블에 민감한 필드가 생기면 다시 검토한다.
- [ ] 현재는 이름/프로필 사진 정도만 노출되는 구조라 급하지 않다.

- [ ] QT 탭 정상 표시

- [ ] 기도 페이지 정상
- [ ] 커뮤니티 페이지 정상
- [ ] 그룹 즐겨찾기 유지
- [ ] PWA 재실행 정상
- [ ] 언어 변경 후 주요 화면 정상
